## 第2周：后端核心 — Express + JWT 认证

### 为什么这个放前面
后端是你最熟悉的领域（因为 MySQL），先学后端能建立信心。而且后端代码只有 10 个文件，比前端简单很多。

### 学习内容

**Day 1-2：Express 基础**

Express 就是一个处理 HTTP 请求的框架。核心概念就三个：

```javascript
// 1. 创建应用
const app = express();

// 2. 定义路由（处理某个 URL 的请求）
app.get('/api/corpus', (req, res) => {
  // req = 请求对象（前端发来的数据）
  // res = 响应对象（你要返回的数据）
  res.json({ success: true, data: [...] });
});

// 3. 启动监听
app.listen(3001, () => console.log('启动了'));
```

对照项目代码，逐个文件看懂：
- `server/index.js`（入口，挂载路由）
- `server/routes/corpus.js`（最纯粹的路由，只有数据库查询）

**Day 3：中间件概念**

Express 的中间件就是"请求到达路由之前，先经过的处理函数"：

```javascript
app.use(express.json());     // 解析 JSON 请求体
app.use(cors(...));           // 允许前端跨域访问
app.use('/api/auth', authRouter);  // 挂载路由
```

看懂 `server/middleware/errorHandler.js`（统一错误处理）。

**Day 4-5：JWT 认证**

这是面试必问的知识点。逐行看懂这两个文件：

- `server/middleware/auth.js` — JWT 验证中间件
- `server/routes/auth.js` — 注册/登录接口

理解流程：

```
注册：用户提交邮箱密码 → bcrypt 加密密码 → 存入数据库 → 返回 JWT token
登录：用户提交邮箱密码 → 查数据库比对 bcrypt → 返回 JWT token
请求：前端请求带 token → auth 中间件验证 token → 通过后执行路由代码
```

**JWT 是什么**：一串加密字符串，包含了用户 ID，7 天后过期。服务端不存 session，完全靠 JWT 验证身份。

**bcrypt 是什么**：密码加密算法。即使数据库被泄露，黑客也解不出原始密码。

**Day 6-7：完整理解后端**

对照 API 列表，逐行看完 `server/routes/` 下的 4 个路由文件。用 curl 或 Postman 手动测试每个接口。

**Day 3-5：CSS 核心 + TailwindCSS 入门**

传统 CSS 你需要写 `.class { color: red; margin: 10px; }`，然后在 HTML 里写 `class="my-class"`。

TailwindCSS 是把样式直接写在 HTML 标签上，不需要单独写 CSS 文件：

```html
<!-- 传统写法 -->

<!-- CSS 文件里 .login-btn { background: blue; color: white; padding: 8px 16px; border-radius: 8px; } -->

<!-- Tailwind 写法（样式直接写在标签上） -->
<button class="bg-blue-600 text-white px-4 py-2 rounded-lg">登录</button>
```

对照项目代码理解常用类名：

| Tailwind 类 | 等价 CSS | 含义 |

| `py-2` | `padding-top:0.5rem; padding-bottom:0.5rem` | 垂直内边距 |
| `rounded-lg` | `border-radius: 0.5rem` | 圆角 |
| `font-bold` | `font-weight: 700` | 加粗 |
| `flex` | `display: flex` | 弹性布局 |
| `grid` | `display: grid` | 网格布局 |
| `mb-4` | `margin-bottom: 1rem` | 下边距 |
| `w-full` | `width: 100%` | 全宽 |
| `shadow-sm` | 轻微阴影 | 卡片感觉 |

## 第4周：TypeScript 入门

### TypeScript 是什么
TypeScript = JavaScript + 类型标注。写代码时告诉电脑"这个变量是字符串""那个参数是数字"，写错时编辑器直接标红提示。

### 学习内容

**Day 1-3：基础类型**

```typescript
// JavaScript（无类型）
function login(email, password) {
  return axios.post('/auth/login', { email, password });
}

// TypeScript（有类型）
function login(email: string, password: string): Promise<Response> {
  return axios.post<ApiResponse<User>>('/auth/login', { email, password });
}
```

核心语法：
- `string`、`number`、`boolean` 基本类型
- `interface` 定义对象形状
- `type` 定义类型别名
- `Array<T>` 数组类型
- `?` 可选属性

**Day 4-5：看懂项目类型定义**

打开 `client/src/types/index.ts`，这就是整个项目的数据结构定义，对应你数据库的 5 张表：

```typescript
// 数据库的 users 表 → 前端的 User 类型
interface User {
  id: number;
  email: string;
  nickname: string;
}
```

**Day 6-7：泛型入门**

```typescript
// ApiResponse<T> 中的 <T> 就是泛型
// 意思是"返回的数据类型不确定，调用时再指定"
interface ApiResponse<T> {
  success: boolean;
  data?: T;        // T 可以是 User、Corpus、任何类型
  message?: string;
}
```

## 第5周：React 核心概念

### React 是什么
React 是一个"用 JavaScript 写 HTML"的库。界面被拆成一个个"组件"，每个组件是一个函数。

### 学习内容

**Day 1-2：组件和 JSX**

```jsx
// 一个最简单的 React 组件
function Hello() {
  const name = 'Alice';
  return <h1>你好，{name}</h1>;  // JSX：在 JS 里写 HTML
}
```

对照 `client/src/pages/Login.tsx`，理解：
- 组件就是一个返回 HTML 的函数
- `{ }` 里面写 JavaScript
- `className` 替代 HTML 的 `class`

**Day 3-4：useState（状态）**

```jsx
function Counter() {
  const [count, setCount] = useState(0);  // 定义一个状态
  return (
    <button onClick={() => setCount(count + 1)}>
      点击了 {count} 次
    </button>
  );
}
```

`useState` 是 React 最核心的 hook。理解"状态变化 → 界面自动刷新"这个模式。

**Day 5-6：useEffect（副作用）**

```jsx
function CorpusList() {
  const [data, setData] = useState([]);

  useEffect(() => {
    // 页面加载时自动执行的代码
    fetch('/api/corpus').then(res => setData(res.data));
  }, []);  // 空数组 = 只在首次加载时执行

  return <div>{/* 渲染 data */}</div>;
}
```

**Day 7：组件通信（props）**

父组件传数据给子组件：
```jsx
// 父组件
<SentenceCard sentence={data} isActive={true} />

// 子组件接收
function SentenceCard({ sentence, isActive }) {
  return <div className={isActive ? 'bg-blue-50' : ''}>{sentence.text}</div>;
}
```

## 第6周：React 进阶

### 学习内容

**Day 1-2：React Router v6（前端路由）**

单页应用的核心技术。URL 变了但页面不刷新：

```jsx
<Routes>
  <Route path="/login" element={<Login />} />     {/* 访问 /login 显示登录页 */}
  <Route path="/corpus" element={<CorpusList />} /> {/* 访问 /corpus 显示语料列表 */}
</Routes>
```

看懂 `client/src/App.tsx` 里的路由配置表。

**Day 3-4：Context + useReducer（全局状态）**

项目用 `AuthContext` 管理登录状态，所有页面都能访问用户信息。

对照 `client/src/context/AuthContext.tsx` 理解：
- `createContext` 创建全局数据
- `useReducer` 管理复杂状态（登录、登出、加载中）
- `Provider` 包裹整个应用

**Day 5：Axios（HTTP 请求）**

```typescript
// 前端发起请求
const res = await axios.post('/api/auth/login', { email, password });
// 后端返回
// { success: true, data: { token: 'xxx', user: {...} } }
```

看懂 `client/src/api/index.ts`：
- 拦截器自动给每个请求加 token
- 401 自动跳转登录页

**Day 6-7：组件组合**

完整的页面组件是如何从零件拼起来的：
- `Layout.tsx`（导航栏） + `<Outlet />`（子页面占位）= 全局布局
- `ProtectedRoute.tsx` = 未登录自动跳转

## 第7周：项目串讲（最重要的一周）

### 目标
把前面学的全部串起来，能讲清楚：用户从打开浏览器到看到数据的完整过程。

### 串讲路线

**Day 1-2：从浏览器到数据库，一个请求的完整旅程**

以"用户登录"为例：

```
1. 用户输入邮箱密码 → 点击登录按钮
2. Login.tsx 调用 authAPI.login()
3. Axios 发送 POST http://localhost:3001/api/auth/login
4. Express 收到请求 → 路由匹配 → auth.js 的 /login 处理函数
5. 处理函数查询数据库：SELECT * FROM users WHERE email = ?
6. bcrypt.compare() 验证密码
7. jwt.sign() 生成 token
8. 返回 { success: true, data: { token, user } }
9. Axios 收到响应 → AuthContext 存储 user 和 token
10. Navigate 跳转到 /corpus
```

## 三个最重要的建议

1. **不要只看，要写。** 每学一个知识点，打开 VSCode 写几行代码验证。光看教程是学不会的。

2. **先跑起来，再理解。** 这个项目已经能跑了。改一个文字颜色、加一个按钮、换一个接口参数，改完看效果，比从头写容易得多。

3. **面试/答辩时能讲清楚比做得多重要。** 与其加 10 个半成品功能，不如把现有的 5 个模块彻底搞懂，能讲出"为什么这样设计""遇到了什么问题怎么解决的"。
